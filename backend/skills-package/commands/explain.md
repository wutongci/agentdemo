# 解释工作流

本命令帮助你理解复杂的代码、概念或技术细节，通过清晰的说明和示例。

## 执行流程

### 1. 理解解释需求

识别用户想要理解的内容：
- 特定代码片段或函数
- 技术概念或设计模式
- 系统架构或工作流程
- 错误信息或问题原因

### 2. 准备解释内容

#### 如果是代码解释
- 使用 `fs_read` 读取相关代码文件
- 分析代码的上下文和依赖关系
- 识别关键的逻辑和数据流

#### 如果是概念解释
- 从基础概念开始
- 逐步深入到复杂细节
- 使用类比和示例帮助理解

### 3. 结构化输出

使用清晰的结构输出解释：

```markdown
## 概述
[一句话总结这是什么]

## 背景知识
[理解这个概念需要的基础知识]

## 详细解释

### 核心原理
[解释基本原理和工作机制]

### 关键组件
- **组件1**：[说明作用和职责]
- **组件2**：[说明作用和职责]

### 工作流程
1. [步骤1]
2. [步骤2]
3. [步骤3]

### 代码示例
```语言
// 简化的示例代码
// 带有详细注释
```

## 常见用途
- [用途1]
- [用途2]

## 常见误区
- [误区1]
- [误区2]

## 相关概念
- [概念1]：[简要说明关系]
- [概念2]：[简要说明关系]

## 进一步学习
[建议的学习资源或方向]
```

### 4. 使用适当的教学技巧

- **自顶向下**：从整体到细节
- **类比**：使用熟悉的概念类比
- **可视化**：使用文字图表或流程描述
- **示例**：提供具体的代码或场景示例
- **对比**：与类似概念对比说明差异

## 使用示例

```
/explain 这个递归函数是如何工作的
/explain 什么是依赖注入
/explain src/utils/cache.go 中的缓存机制
/explain 为什么会出现这个错误
```

## 注意事项

- 调整解释的深度和复杂度
- 避免使用过多术语，或者解释所用术语
- 提供实际示例而非抽象描述
- 循序渐进，不要一次性堆砌太多信息
- 确认理解程度，准备回答后续问题
